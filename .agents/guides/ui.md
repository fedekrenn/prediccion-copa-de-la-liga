# UI y estilos

- Usa siempre Tailwind CSS para estilos.
- Iconos de `tabler-icons` con importacion explicita; no usar barrels.
- No duplicar clases si se puede extraer un componente.
- Priorizar legibilidad frente a micro-optimizaciones visuales.
- Accesibilidad no es opcional: HTML semantico, roles ARIA cuando aplique y foco gestionado.
